How It Works
  1. Admin generates an RSA-2048 key pair (once)
  2. Institution fills in student certificate details
  3. System signs the data with the private key
  4. A QR code embedding the signed payload is generated
  5. Certificate image is produced with the QR code embedded
  6. Anyone can scan the QR code to verify authenticity

Security Architecture
🔏
RSA-2048 Signing

Certificate data is digitally signed using RSA with PSS padding and SHA-256 hashing. Only the institution's private key can create a valid signature.

📱
QR Embedding

The signed payload — certificate data plus signature — is encoded into a QR code attached to every issued certificate.

✅
Public Verification

Anyone can verify a certificate by entering its ID or scanning the QR code. No login required. Tampering is detected instantly.
