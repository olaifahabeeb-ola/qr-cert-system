What Happens When You Issue
  1. A unique Certificate ID is assigned
  2. Certificate data is serialised to canonical JSON
  3. RSA-2048 + SHA-256 digital signature is computed
  4. Signed payload is encoded into a QR code image
  5. A printable certificate image is generated
  6. All data is stored securely in the system
⚠️ Important Notes
  • Ensure all details are correct before issuing — certificates cannot be edited after signing.
  • The private key must remain confidential at all times.
  • The public key is used for all verifications and can be shared freely.
  • If keys are regenerated, old certificates will fail verification.
📸 Photo Requirements
  • Clear passport-sized photograph
  • Plain white or light background
  • Face must be clearly visible
  • JPG or PNG format only
  • Recommended size: 300x400 pixels
